# IPTV PRO V52 - GitHub Pages Deployment Guide

## 🎯 সমস্যা এবং সমাধান

### সমস্যা: GitHub Pages-এ HTTP চ্যানেল প্লে হচ্ছে না
- **কারণ:** GitHub Pages HTTPS ব্যবহার করে, কিন্তু অনেক চ্যানেল HTTP লিঙ্ক ব্যবহার করে
- **ফলাফল:** ব্রাউজার Mixed Content নীতির কারণে HTTP স্ট্রিম ব্লক করে দেয়

### সমাধান: Advanced Proxy Mechanism
আমরা একটি স্মার্ট প্রক্সি সিস্টেম যোগ করেছি যা:
1. **স্বয়ংক্রিয় Mixed Content সনাক্তকরণ** - HTTPS পেজ + HTTP স্ট্রিম শনাক্ত করে
2. **বুদ্ধিমান প্রক্সি নির্বাচন** - ফাইলের ধরন অনুযায়ী সেরা প্রক্সি বেছে নেয়
3. **স্বয়ংক্রিয় ফেইলওভার** - প্রথম প্রক্সি ব্যর্থ হলে পরবর্তী চেষ্টা করে
4. **নেটওয়ার্ক অপ্টিমাইজেশন** - স্লো নেটওয়ার্কের জন্য বিশেষ কনফিগারেশন

## 🚀 GitHub Pages-এ ডিপ্লয় করার ধাপ

### ধাপ ১: ফাইল প্রতিস্থাপন
```bash
# player.js এর স্থানে player-final.js ব্যবহার করুন
cp player-final.js player.js

# app.js এর স্থানে app-final.js ব্যবহার করুন
cp app-final.js app.js
```

### ধাপ ২: HTML ফাইল আপডেট করুন
`player.html` এবং `index.html` এ নিম্নলিখিত পরিবর্তন করুন:

**player.html-এ:**
```html
<!-- পুরানো লাইন -->
<script src="player.js?v=45"></script>

<!-- নতুন লাইন -->
<script src="player.js?v=52"></script>
```

**index.html-এ:**
```html
<!-- পুরানো লাইন -->
<script src="app.js?v=50"></script>

<!-- নতুন লাইন -->
<script src="app.js?v=52"></script>
```

### ধাপ ৩: Git-এ পুশ করুন
```bash
git add .
git commit -m "Update: V52 - Advanced Proxy System for GitHub Pages HTTPS"
git push origin main
```

### ধাপ ৪: ক্যাশ ক্লিয়ার করুন
GitHub Pages ক্যাশ পরিষ্কার করতে:
1. Repository Settings → Pages
2. Source পরিবর্তন করুন এবং আবার ফিরিয়ে আনুন
3. বা ব্রাউজারে Ctrl+Shift+Delete দিয়ে ক্যাশ ক্লিয়ার করুন

## 🔧 প্রক্সি সিস্টেম কীভাবে কাজ করে

### প্রক্সি লিস্ট (অগ্রাধিকার অনুযায়ী)
```javascript
1. Direct (শুধু HTTPS স্ট্রিমের জন্য)
2. corsproxy.io (বাইনারি .ts ফাইলের জন্য সেরা)
3. api.allorigins.win (M3U8 এবং JSON-এর জন্য)
4. api.codetabs.com (নির্ভরযোগ্য ফলব্যাক)
5. thingproxy.freeboard.io (বিকল্প)
6. proxy.cors.sh (শেষ অবলম্বন)
7. cors-anywhere.herokuapp.com (জরুরি ফলব্যাক)
```

### স্বয়ংক্রিয় সিদ্ধান্ত গ্রহণ প্রক্রিয়া
```
1. URL বিশ্লেষণ করুন (HTTP বনাম HTTPS)
2. পেজ প্রোটোকল চেক করুন (HTTPS পেজ?)
3. Mixed Content সনাক্ত করুন (HTTPS + HTTP?)
4. ফাইল ধরন চেক করুন (.ts, .m3u8, .mpd?)
5. সেরা প্রক্সি নির্বাচন করুন
6. প্লেব্যাক ইঞ্জিন চেষ্টা করুন (HLS.js → Native → Clappr → Shaka)
7. ব্যর্থ হলে পরবর্তী প্রক্সি চেষ্টা করুন
```

## 🧪 পরীক্ষা করার ধাপ

### লোকাল পরীক্ষা
```bash
# লোকাল সার্ভার চালু করুন
python3 -m http.server 8000

# ব্রাউজারে খুলুন
http://localhost:8000
```

### GitHub Pages পরীক্ষা
1. আপনার GitHub Pages URL খুলুন
2. একটি HTTP চ্যানেল নির্বাচন করুন
3. ব্রাউজার কনসোল খুলুন (F12)
4. লগ দেখুন - "Mixed content detected..." বার্তা দেখা উচিত
5. চ্যানেল প্লে হওয়া উচিত প্রক্সির মাধ্যমে

### ডিবাগিং কনসোল লগ
```
Pro Player V52: Loading stream: http://...
Mixed content detected (HTTPS page, HTTP stream). Forcing proxy...
Using Proxy 1: https://corsproxy.io/?url=...
Attempt 1/35: Engine=hls, Proxy=1, URL=http://...
HLS Manifest parsed successfully
```

## ⚠️ সম্ভাব্য সমস্যা এবং সমাধান

### সমস্যা ১: চ্যানেল এখনও প্লে হচ্ছে না
**সমাধান:**
- ব্রাউজার ক্যাশ ক্লিয়ার করুন (Ctrl+Shift+Delete)
- GitHub Pages ক্যাশ ক্লিয়ার করুন (Settings → Pages → Source পরিবর্তন)
- চ্যানেল লিঙ্ক অনলাইন আছে কিনা যাচাই করুন
- কনসোল লগ দেখুন সঠিক প্রক্সি ব্যবহার হচ্ছে কিনা

### সমস্যা ২: প্রক্সি খুব ধীর
**সমাধান:**
- স্লো নেটওয়ার্ক মোড সক্ষম আছে কিনা চেক করুন
- বিভিন্ন চ্যানেল চেষ্টা করুন
- সার্ভার লোড কমলে আবার চেষ্টা করুন

### সমস্যা ৩: কিছু চ্যানেল শুধু লোকালে কাজ করে
**কারণ:** লোকালে HTTP সরাসরি কাজ করে, কিন্তু GitHub Pages-এ HTTPS প্রয়োজন
**সমাধান:** এটি স্বাভাবিক, প্রক্সি সিস্টেম এটি সামলায়

// স্বয়ংক্রিয়ভাবে মোবাইল নেটওয়ার্ক ধরা পড়ে:

## 🔐 নিরাপত্তা নোট

### CORS এবং Mixed Content নীতি
- সমস্ত প্রক্সি CORS সক্ষম
- Mixed Content স্বয়ংক্রিয়ভাবে পরিচালিত হয়
- কোন ব্যক্তিগত তথ্য প্রক্সির মাধ্যমে পাঠানো হয় না

### প্রক্সি সার্ভার
- সমস্ত প্রক্সি জনসাধারণের জন্য উপলব্ধ
- কোন লগিং বা ট্র্যাকিং নেই
- বিনামূল্যে এবং ওপেন সোর্স

## 📞 সহায়তা এবং ডিবাগিং

### ব্রাউজার ডেভেলপার কনসোল
```javascript
// কনসোল খুলুন: F12 বা Ctrl+Shift+I
// লগ দেখুন:
console.log("Pro Player V52: Loading stream...")
console.warn("Mixed content detected...")
console.error("HLS Error: ...")
```

### নেটওয়ার্ক ট্যাব
1. F12 খুলুন → Network ট্যাব
2. চ্যানেল প্লে করুন
3. প্রক্সি অনুরোধ দেখুন
4. স্ট্যাটাস কোড চেক করুন (200 = সফল)

### সাধারণ ত্রুটি কোড
- `200` - সফল
- `404` - চ্যানেল খুঁজে পাওয়া যায়নি
- `403` - অ্যাক্সেস অনুমতি নেই
- `500` - সার্ভার ত্রুটি
- `CORS Error` - প্রক্সি ব্যর্থ
